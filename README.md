# UART-CPU

### 4. CPU_L_test - Процессорное ядро

RISC-подобное процессорное ядро с 4-стадийным конвейером для выполнения команд управления LED.

**Параметры:**
- `CMD_MEM_SIZE` (default: 8) - размер памяти команд
- `CMD_SIZE` (default: 16) - размер команды в битах
- `MEM_SIZE` (default: 16) - размер памяти данных
- `LIT_SIZE` (default: 8) - размер литерала в битах
- `RF_SIZE` (default: 8) - количество регистров
- `COP_SIZE` (default: 4) - размер кода операции

**Входы:**
- `clk` - тактовый сигнал
- `reset` - сброс
- `cmd_out[31:0]` - входная команда от парсера
- `data_out[47:0]` - входные данные от парсера
- `gotov` - сигнал готовности команды

**Выходы:**
- `LED[3:0]` - состояние светодиодов
- `vovan` - отладочный сигнал

**Регистровый файл:**
- 8 регистров (R0-R7) разрядностью 12 бит
- Регистр R0 всегда равен 0

**Память данных:**
- 16 ячеек по 12 бит
- Загружается из файла `mem.mem`

**Память команд:**
- 8 ячеек по 16 бит
- Загружается из файла `cmd_mem.mem`

**Принцип работы:**
- Имеет 5 стадий конвейера (0-4)
- Стадия 0: выборка команды из памяти команд
- Стадия 1: чтение первого операнда из регистров или памяти
- Стадия 2: чтение второго операнда
- Стадия 3: выполнение арифметико-логической операции
- Стадия 4: запись результата и обновление PC
- Запускается при получении по UART команды "0000"

### 5. ascii_t - Интерпретатор LED команд

Модуль для преобразования команды в состояние светодиодов.

**Параметры:**
- `N` (default: 8) - общее количество 4-битных значений
- `CMD_M` (default: 4) - количество 4-битных значений в команде

**Входы:**
- `clk` - тактовый сигнал
- `ready` - сигнал готовности команды
- `data_in[((N-CMD_M)*4)-1:0]` - входные данные (16 бит)
- `cmd_in[CMD_M*8-1:0]` - входная команда (32 бита)

**Выходы:**
- `LED[3:0]` - состояние светодиодов
- `gotov` - сигнал готовности

**Принцип работы:**
- Проверяет команду на соответствие "LED " (0x4C4544)
- При совпадении интерпретирует следующие 4 hex цифры как состояние LED
- Каждая hex цифра (0-F) управляет одним светодиодом (0 - выкл, 1-F - вкл)
- Формирует импульс `gotov` после обработки

### 6. out_manager - Менеджер исходящих сообщений

Модуль для формирования адресов ответных сообщений в памяти.

**Входы:**
- `clk` - тактовый сигнал
- `TX_b` - сигнал занятости UART передатчика
- `cmd_in[2:0]` - код команды
- `data_in[7:0]` - состояние LED
- `ready` - сигнал готовности к отправке

**Выходы:**
- `data_out[15:0]` - адрес для чтения из памяти
- `gotov` - сигнал готовности для UART_TX

**Принцип работы:**
- Ожидает освобождения UART передатчика (TX_b == 0)
- Формирует последовательность адресов для отправки сообщений о состоянии LED
- Генерирует сигнал `gotov` для запуска UART_TX

### 7. mem - Память сообщений

Модуль для хранения ASCII-строк ответных сообщений.

**Входы:**
- `clk` - тактовый сигнал
- `adr[15:0]` - адрес для чтения

**Выходы:**
- `data_out[7:0]` - байт данных из памяти

**Содержимое памяти:**
| Адрес | Сообщение | Описание |
|-------|-----------|----------|
| 0x0001-0x000A | "LED 1 ON\r\n" | LED1 включён |
| 0x0011-0x001A | "LED 2 ON\r\n" | LED2 включён |
| 0x0021-0x002A | "LED 3 ON\r\n" | LED3 включён |
| 0x0031-0x003A | "LED 4 ON\r\n" | LED4 включён |
| 0x0101-0x010B | "LED 1 OFF\r\n" | LED1 выключен |
| 0x0111-0x011B | "LED 2 OFF\r\n" | LED2 выключен |
| 0x0121-0x012B | "LED 3 OFF\r\n" | LED3 выключен |
| 0x0131-0x013B | "LED 4 OFF\r\n" | LED4 выключен |

### 8. top_m - Верхнеуровневый модуль (только приём)

Упрощённая версия системы только для приёма команд и управления LED.

**Входы:**
- `CLK` - тактовый сигнал
- `RESET` - сброс
- `UART_RXD` - вход UART

**Выходы:**
- `LED[3:0]` - состояние светодиодов

### 9. MM_top - Полная система

Объединяет все модули в законченное устройство с приёмом, обработкой и передачей ответов.

**Входы:**
- `CLK` - тактовый сигнал
- `RESET` - сброс
- `UART_RXD` - вход UART

**Выходы:**
- `UART_TXD` - выход UART
- `LED[3:0]` - состояние светодиодов

## Система команд процессора

| Код | Мнемоника | Описание | Формат команды |
|-----|-----------|----------|----------------|
| 0000 | NOP | Нет операции | `[0000][...]` |
| 0001 | ADD | R3 = R1 + R2 | `[0001][R1][R2][R3]` |
| 0010 | SUB | R3 = R1 - R2 | `[0010][R1][R2][R3]` |
| 0011 | MOVI | R1 = literal | `[0011][R1][literal]` |
| 0100 | MOVR | R3 = R1 | `[0100][R1][R3]` |
| 0101 | LOAD | R3 = mem[addr] | `[0101][addr][R3]` |
| 0110 | STORE | mem[addr] = R1 | `[0110][addr][R1]` |
| 0111 | JMP | pc = addr | `[0111][addr]` |
| 1000 | JZ | if(R1==0) pc=addr | `[1000][R1][addr]` |
| 1001 | JNZ | if(R1!=0) pc=addr | `[1001][R1][addr]` |
| 1010 | CMP | Сравнение R1 и R2 | `[1010][R1][R2]` |
| 1011 | INC | R1 = R1 + 1 | `[1011][R1]` |
| 1100 | DEC | R1 = R1 - 1 | `[1100][R1]` |
| 1101 | AND | R3 = R1 & R2 | `[1101][R1][R2][R3]` |
| 1110 | LD | LED = R1[3:0] | `[1110][R1]` |
| 1111 | END | Останов процессора | `[1111]` |

## Тестовые стенды

### tb_CPU_L_test
Тестирует работу процессорного ядра:
- Загружает тестовую программу из `cmd_mem.mem`
- Проверяет выполнение инструкций
- Генерирует VCD-файл для анализа временных диаграмм

### tb_m1
Тестирует цепочку out_manager1 → mem → UART_TX → UART_RX:
- Проверяет формирование ответных сообщений
- Тестирует отправку и приём данных

### tb_top
Интеграционный тест всей системы:
- Отправляет команду "0000" для запуска процессора
- Проверяет взаимодействие всех модулей
